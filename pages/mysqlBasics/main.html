Why do we need relational databases?

  • Storing data is not the problem, managing it however is a tougher task.

  • We need to retrieve stored information quickly.

  • We want to form relationships between our datasets.

  • We want to update our stored information quickly and safely.

What is a relational database?

  • A database consists of tables. You can store up to 2,147,483,647 tables in a single database.

  • Each table consists of records(rows) and fields(columns).

  • The rows are split into columns, which contain our stored data.

  • Most tables poses a Primary Key(PK). Tables can not contain duplicate Primary Keys.

  • Tables can also poses a Foreign Key(FK). Foreign Keys(FK) are used to create relationships between tables.

  • There are three kinds of relationships between tables - one-to-one, one-to-many and many-to-many.

  • In a one-to-one relationship a row from table "A" can be linked to exactly one row from table "B", and a row from database "B" can be linked with exactly one row from table "A".

  • In a one-to-many relationship a row from table "A" can be linked with an arbitrary amount of rows from table "B", but a row from table "B" can not be linked with more than one row from table "A".

  • In a many-to-many relationship a row from table "A" can be linked with an arbitrary amount of rows from table "B" and vice versa.

  • In order to represent the relationships between tables in a database, we use UML diagrams.

How do we use a relational database?

  • In order to use a relational database, we need a Relational Database Management System. RDBMS


  • The RDBMS is a specialized software solution for manipulating data in a relational database.

  • Some of the most popular RDBMS are Oracle, MySQL, MS SQL Server and PostgreSQL.

  • In order to manipulate the data stored in our database, we give the RDBMS instructions using a special declarative programming language named SEQUEL(SQL).


  • By using SQL we can create, retrieve, update and delete data from our relational database. This set of operations is known as CRUD.
